1er Parcial

Diagramas de flujo y Pseudocódigo

Empezamos el primer parcial viendo diagramas de flujo. Para entenderlos es importante conocer la simbología que los compone.

Los diagramas de flujo permiten representar de forma gráfica los pasos que sigue un algoritmo para resolver un problema. Cada símbolo tiene un significado específico, lo que facilita la comprensión de la lógica del proceso.

Gracias a los diagramas de flujo aprendimos a identificar entradas, procesos y salidas, lo cual es fundamental antes de comenzar a programar en cualquier lenguaje.

Estas prácticas nos ayudaron a desarrollar el pensamiento lógico y a ordenar correctamente las instrucciones necesarias para que un programa funcione de manera adecuada.

Pseudocódigos

El pseudocódigo utiliza un lenguaje sencillo, parecido al lenguaje humano, lo que permite enfocarse en la lógica del problema sin preocuparse por la sintaxis de un lenguaje de programación real.

Con la herramienta PSeInt aprendimos a escribir algoritmos, usar estructuras condicionales como SI, ciclos como MIENTRAS y variables para almacenar datos.

Estas actividades fortalecieron nuestra comprensión de la lógica computacional y nos prepararon para programar posteriormente en lenguajes más avanzados.

Metodología de desarrollo de software

La metodología de desarrollo de software es un conjunto de pasos, reglas y técnicas que se utilizan para planear, diseñar, construir, probar y mantener un sistema de software.

Su objetivo principal es organizar el trabajo, reducir errores y asegurar que el software cumpla con las necesidades del usuario final.

Gracias a las metodologías, los equipos de desarrollo pueden trabajar de forma ordenada, cumplir tiempos de entrega y mejorar la calidad del producto.

Tipos de usuario

Son categorías de personas que interactúan con un sistema.

Los tipos de usuario permiten definir qué acciones puede realizar cada persona dentro de un sistema, ayudando a mantener la seguridad y el correcto funcionamiento del software.

  • Administrador: Tiene control total del sistema.
  • Usuario estándar: Puede usar las funciones principales.
  • Invitado: Solo puede visualizar información.

Identificar los tipos de usuario es importante para evitar accesos no autorizados y garantizar que cada persona use el sistema de manera correcta.

2do Parcial

Diseño Punto de venta

En el segundo parcial de esta materia vimos el diseño de cómo realizar un punto de venta. Aprendimos cómo se compone y qué elementos debe llevar, dependiendo de las necesidades del cliente.

El sistema debía contar con dos secciones principales: Usuario y Tipo de Usuario. En estas secciones se registraban los datos del usuario, como nombre completo y demás información general, así como la función que desempeñaba dentro del sistema.

Los tipos de usuario podían ser gerente, administrador, cajero o empleado. Cada uno contaba con diferentes permisos y alcances dentro del programa, ya que no todos podían acceder a las mismas funciones.

Para el diseño del punto de venta utilizamos la aplicación Balsamiq, la cual nos permitió realizar un maquetado más claro, ordenado y detallado antes de comenzar con la programación.

3er Parcial

Bases de Datos

Una base de datos es un conjunto organizado de información que se almacena de forma estructurada para facilitar su acceso, consulta y actualización. Se utilizan para guardar datos de usuarios, productos, ventas, calificaciones y muchos otros elementos.

Las bases de datos permiten trabajar con grandes cantidades de información de manera segura y eficiente. Son fundamentales en sistemas como puntos de venta, aplicaciones web y sistemas escolares.

Algunos ejemplos de gestores de bases de datos son:

  • MySQL
  • SQL Server
  • Oracle
  • PostgreSQL

¿Qué es la programación orientada a objetos?

La programación orientada a objetos (POO) es un paradigma de programación que organiza el software en objetos. Cada objeto representa algo del mundo real y contiene datos y acciones.

La POO se basa en cuatro principios fundamentales:

  • Encapsulamiento: protege los datos del objeto.
  • Herencia: permite reutilizar código.
  • Polimorfismo: un objeto puede comportarse de diferentes formas.
  • Abstracción: muestra solo lo necesario.
